
Техническое задание: Аналитическая модель пневматической системы распыления

1. Контекст системы

Объект — промышленная установка дезоксидации внутренней поверхности стальных
гильз. Процесс заключается во вдувании порошкообразного дезоксиданта потоком
азота (N_2) через сопло.

  - Рабочая среда: Азот и мелкодисперсный порошок.
  - Длительность цикла: От 3 до 7 секунд (весь процесс протекает в динамике,
    установившихся режимов нет).
  - Исполнительные механизмы: Пневматические клапаны с усилителями, управляемые
    по положению (0–100%) без прямой обратной связи по положению.
  - Контроль: Манометры на линии подачи азота в транспортную трубу и в бак с
    порошком (P-1). Тензодатчики веса бака (для контроля расхода порошка).

2. Логика управления (Входные данные)

Оператор задает кривую давления через 10 точек (фаз):

  - Давления (P_1 \dots P_{10}): Целевые значения для каждой фазы.
  - Временные интервалы (\%_1 \dots \%_8): Процентное распределение общего
    времени впрыска (T_{injection}) между фазами.
  - Система управления: Превращает эти точки в ступенчатый сигнал управления
    клапанами.

3. Проблема

Существует значительный разрыв между «заданной настройкой» и «реальным
давлением» в системе из-за:

1.  Инерции пневмоусилителей и объемов магистралей/баков.
2.  Запаздывания (Latency) срабатывания механизмов.
3.  Нелинейности характеристик клапанов.
4.  Взаимовлияния потоков (эффект эжекции/Вентури в узле смешения).

4. Цель разработки

Создать программную модель, решающую две задачи:

  - Прямая задача: На основе 10 точек настройки (массивы давлений и времени)
    спрогнозировать непрерывные графики реального давления в баке и трубе.
  - Обратная задача (перспектива): Рекомендовать настройки (10 точек) для
    достижения желаемого профиля расхода азота и порошка.

5. Выбранный метод моделирования

Отказаться от чистых нейросетей в пользу гибридной физико-математической модели
(Grey Box):

  - Использовать передаточные функции первого или второго порядка
    (дифференциальные уравнения) для описания динамики давления.
  - Базовое уравнение: \tau \frac{dP}{dt} + P = K \cdot S(t - L), где:
      - P — прогнозируемое давление.
      - S — ступенчатая настройка из ЧМИ.
      - K — коэффициент усиления (перевод % в бары).
      - \tau — постоянная времени (инерция).
      - L — чистое запаздывание.
  - Параметры K, \tau, L могут быть нелинейными функциями от текущего давления
    или настроек.

6. Требования к реализации (Этап 1: Давления)

1.  Алгоритм расчета: Реализовать итерационный цикл (шаг dt \approx 0.01 с),
    который «проигрывает» весь цикл впрыска.
2.  Вход: Массив настроек (10 давлений, 8 временных долей, общее время).
3.  Обработка: Модель должна учитывать, что давление в начале новой фазы равно
    давлению в конце предыдущей (непрерывность).
4.  Калибровка: Предусмотреть возможность подбора коэффициентов K, \tau, L на
    основе имеющихся исторических логов (CSV/Excel с реальными графиками).

7. Ключевые физические нюансы для учета

  - Эффект Вентури: Поток порошка возможен даже при P_{бака} < P_{трубы}, так
    как в узле смешения создается локальное разряжение.
  - Переходные процессы: Модель должна адекватно описывать «недолет» давления до
    уставки, если фаза слишком короткая.

11. Математическое ядро модели (Logic Kernel)
В промтах для описания физики используем формат уравнений в коде (так ИИ понимает зависимости лучше):
1. Динамика привода клапана (1-й порядок):
code
Python
# Изменение положения клапана за шаг времени dt
dv = (target_signal - v_pos) / (T_valve_time + 1e-6)
v_pos = v_pos + dv * dt
2. Динамика давления (2-й порядок):
code
Python
# Расчет ускорения давления (вторая производная)
d2P = (K_gain * v_pos - 2 * damping * T_const * dP - P) / (T_const ** 2)

# Интегрирование (переход к скорости и давлению)
dP = dP + d2P * dt
P = P + dP * dt
3. Нелинейная коррекция (Sharpness):
code
Python
# Разрыв между целью и фактом
gap = abs(target_p - current_p)

# Динамическая резкость: чем больше gap, тем выше current_sharp
current_sharp = min(sharp_base + (gap * sharp_sens), sharp_max)

# Итоговый прогноз с учетом импульса dP
p_result = P + (current_sharp * T_const * dP)
12. Логика оценки точности (MAE Calculation)
Алгоритм расчета ошибки MAE (Mean Absolute Error) оптимизирован для промышленных графиков:
Определение рабочей зоны: Система ищет индексы trigger_open и trigger_close в логах.
Изоляция: Если клапаны срабатывали, MAE считается только внутри этого интервала. Это критично, так как модель не должна штрафоваться за "шумы" в баке, когда впрыск не идет.
Остаточная ошибка (Residual): В каждой точке создается колонка p_residual_error = (Fact_P - Theory_P).
13. Список ключевых параметров (flat_params)
Для передачи в промт используйте этот список названий (они соответствуют коду):
T_k_gain — коэффициент усиления (сколько бар выдает клапан на 100% открытия).
T_time_const — инерция бака и магистралей.
T_damping — гашение колебаний (демпфирование).
T_dead_time — чистое запаздывание (транспортная задержка).
T_valve_time — время раскрутки/хода самого клапана.
T_sharp_base, T_sharp_sens, T_sharp_max — параметры "резкости" для имитации работы пневмоусилителей.
